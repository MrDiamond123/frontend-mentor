# AGENTS.md

## Overview

This repository contains my solutions to challenges from
[Frontend Mentor](https://www.frontendmentor.io/). Challenges are organized
into folders by difficulty level.

## Root Project

The root of this repository has **no constraints**. It is not itself a
Frontend Mentor challenge — treat it as a general workspace. Create tooling,
scripts, shared assets, documentation, or anything else the user asks for at
the root level without restrictions.

## Constraint Hierarchy

**Each difficulty folder contains its own `AGENTS.md`, copied from Frontend
Mentor, that applies to every challenge inside it.**

When working on anything, follow this precedence (most specific wins):

1. `AGENTS.md` inside the individual challenge folder (if present)
2. `AGENTS.md` in the difficulty folder (copied from Frontend Mentor — always
   applies)
3. This root `AGENTS.md`

The difficulty-level `AGENTS.md` files define the rules for that tier (e.g.
allowed technologies, required approach). **Never override or contradict
them** when working inside that difficulty folder — they take precedence over
anything in this file.

## Working in This Repo

- **New challenge:** scaffold it in the correct difficulty folder and follow
  that folder's `AGENTS.md` from the start.
- **Existing challenge:** read the difficulty folder's `AGENTS.md` first, then
  any challenge-level `AGENTS.md`, before making changes.
- **Root-level work** (tooling, scripts, docs, experiments): no constraints —
  just do what the user asks.
- Keep challenges self-contained and don't commit build output or
  `node_modules`.
